## 裁定

**1. Round 1 的「turn 膨胀」正式撤回——是 n=1 方差。**
R1-C 在 n=3 下 turns 中位 18 vs A 的 36,成本中位为 A 的 **33%**,
三次 static 全对。上一轮 ×2.5 的刺眼数字被同臂重复直接反转。
判定纪律(不落终判)避免了一次方向性错误结论,记入方法论。

**2. 优势区命题部分成立,且边界比假设更清晰。**
- **Refactor(code-production 重)**:成立。C 全部三次 run 落在
  A 的最低值附近或之下,配对秩上 C 占优(仅一处交叠 857 vs 492)。
  这与原始假设(曲线 B:工作产物压缩)一致——写代码产生大 args,
  压缩它们直接省钱且不伤 acceptance。
- **Exploration**:不成立。C 成本与 A 持平(1616 vs 1527),
  且 completion 不稳(三次里一次 0/5)——但注意 **A 自己也一次 0/5**:
  探索类任务的「做丢产物」是任务性质,不是压缩独有病。
- **负区(direct-transform)**:未咬人。8k 门下投影最多 43 次,
  C 三次 static 全对、成本中位还低于 A。在本探针规模下,
  taucode 时代的负区恐惧未复现——要么保真够用,要么探针还不够狠。

**3. E1-D 的完成稳定性是全数据集最干净的信号。**
两轮累计 **6/6 次 5/5**(round1 一次 + round2 三次),四臂唯一。
代价:成本最高(2445 中位,约 A 的 1.6 倍)。round 1 的「省 46%」
未复现(运行级方差),但完成率的复现性完好。裁定表述:
**hybrid+checkpoint 买的是探索类任务的完成率,不是成本**。
机制猜想(待 C'' 检验):checkpoint 的持久摘要给了模型跨压缩的
进度锚——这正是 V3-WS 语义锚点想以零 LLM 成本做到的事。

**4. B'(native LLM-summary,可比阈值)是体面的对手。**
R1 601(次于 C),E1 **1287(四臂成本最优)**且 completion 尚可
(4/5,5/5,5/5)。但 D1 触发率 1/3——native 的阈值语义
(context-window 挤压)在小任务上不可靠,作为策略它的可控性弱于
显式门控。公平记录:在 exploration 上,LLM 摘要的语义融合能力
可能真的优于纯 extractive——这与 Factory 的评估方向一致。

## 诚实边界

- n=3,配对秩可读方向,不支持显著性宣称;
- 质量仅到 static acceptance,**产物盲评未做**(artifact 已留存,
  人工审计待做——尤其 R1-C 三次拆分的代码质量);
- 单模型(v4-flash)单 harness,外推需谨慎;
- E1-D round1「省 46%」与 round2「贵 60%」并存——运行级成本方差
  仍大于策略间差异,除 R1-C 外的成本结论都应加宽置信。
