The current sideband implementation is too chatty: it summarizes many compacted read results, repeats aggressively, and allows summaries large enough to dominate output cost. The extra semantic surface did not improve completion stability in this batch.

## Verdict

Branch B does not promote `C-SB` as implemented. It is mechanically active, but the measured form is costly and unstable: 1/3 success, high sideband output, and no evidence that substitution improves completion over anchor/placebo controls.

`C+PL` is the only 3/3 arm in this batch. This should not be narrated as "placebo content is useful" or "anchors are bad". The narrower inference is that a projected-turn tail affordance may be acting as a useful self-orientation cue independent of semantic content. That points toward testing lightweight, explicit state/nudge designs before expanding semantic substitution.

## First-Principles Fixes

Before retrying `C-SB`, the sideband producer needs tighter invariants:

- Dedupe by `path#hash`: one summary per stable source view, not one summary per repeated projection diff.
- Bound output hard: use a structured compact schema with a strict max token target; current prose can become larger than the value it replaces.
- Gate summarization by value: summarize only views that are likely to be re-used, not every compacted read result.
- Coalesce per projection: cap sideband calls per projected turn and per run.
- Substitute only calibrated summaries: summary records should earn trust through non-reread evidence or explicit model declaration, not by existing.

Next decision point:

- Keep `README` narrative frozen.
- Treat `C+PL 3/3` as a hypothesis for "tail affordance / self-anchor" and use it to prioritize a lightweight nudge or explicit-state branch.
- Do not run Branch B production-style `C-SB` again until dedupe and budget caps are implemented.
