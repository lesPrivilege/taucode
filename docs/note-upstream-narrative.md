2026-07-05。状态:**预验证叙事**——机制已野外确认,经济学结论待 G2 配对

## 一句话

一个绑定单一 backend（DeepSeek）的开源验证台:在最薄的 harness（pi）上,
把 compaction、cache、信任协议做成**可观测、可复现、可证伪**的实验对象——
下游做不了权重,但可以把「权重该内化什么」测量清楚,交给能内化它的人。

## 我们替上游测量了什么（核心价值主张）

**1. 成本结构是双模态分布,不是一个数。** 同一套确定性 compaction,
在干净读分析上成本表现为 cache-break（触发即 CH dip,零重读）,在歧义
编辑上表现为重读/自证螺旋（CH 崩 + reasoning 暴涨)。对定价与 cache
设计的含义:hit 折扣的实际兑现率是任务类型的函数——单一费率下,两类
工作负载的真实成本差可达数量级。[G2 数字回填位]

**2. (d) 类自证螺旋:一个可机械判别的行为失效。** 判别器:
`re_reads > 0` 且 `compacted_path_re_reads ≈ 0`(确证缺口,非信息缺口),
伴随 CH 崩 + reasoning 激增。我们的 harness 级解法(hash 溯源的
stale-view hint)野外验证有效——模型看到因果锚点即停止怀疑,时序推理
精确(区分自编辑/跨文件/hash 匹配)。**但持久解在权重**:模型对自身
工作产物的信任校准是能力问题,harness 只能补偿。这正是可交给 RL 的
失效分类学:(a) 理性恢复 /(b) 摘要缺陷 /(c) 循环病理 /(d) 确证螺旋,
各自的行为指纹都已可从 telemetry 机械提取。

**3. 校准指标 = 能力等高线的实测点。**「声明可丢弃但后来重读」率
测量模型对自身信息需求的自知程度。跨模型/跨代际跑同一套,差值就是
一条能力等高线——这是下游能产出、只有上游能消费的信号。

**4. 意图声明作为 API 面的提案。** tool call 携带 retention 语义
(`semantic | verbatim | routing`)让 compaction 从 harness 猜测变成
模型自述。若上游把它做成 tool-calling 协议的一等公民(类比
reasoning_content 的处理),整个下游生态的 context 经济都受益——
且声明质量本身可训练。

## 对 DeepSeek 的具体话

- 你们的 `prompt_cache_hit_tokens` 是**整个项目可行的前提**——一个字段
  让下游第一次能对账「预期 transition 曲线 vs 实测」。同行里这不是标配。
- 求更进一步:cache 命中的**边界诊断**(前缀在第几 token 断的)。现在
  下游只能从 hit 比例反推断点,一个字段的事,能让所有 harness 的
  cache 调试从玄学变工程。
- session 重放的 reasoning_content 处理(#3636 修复路径)是「上游内化
  补偿」的教科书案例——我们核对时连预设的失效方向都错了,这值得写进
  你们自己的叙事。
- server-side compaction 的机会:我们的确定性投影证明压缩区可以重新
  字节稳定;若压缩发生在 cache 契约之内(server 端),transition 成本
  直接归零。参照系已有(Anthropic compact-2026-01-12),DeepSeek 的
  价差结构下这件事的杠杆更大。

## 会计学框架（论文论点,一段话版）

下游做 harness,折旧是净损失;上游做,折旧的同时拿三笔:训练信号
(我们的失效分类学与 telemetry)、能力等高线(校准指标)、跨层 cache
契约(意图声明 + server-side compaction)。本项目把第一、二笔打包
成了可直接消费的格式——**我们的折旧,明码标价,是你们的信号**。
