# 确定性 compaction 的一次实作记录

2026-07-05

Agent loop 的成本结构里，input 远大于 output——多轮工具调用下 100:1
是常见量级，所以 prefix cache 的命中率几乎决定了单位成本。provider 侧
的定价也围绕这一点：cached input 与 uncached input 的价差在 10 倍上下，
DeepSeek 会在 usage 里报告 prompt_cache_hit_tokens，这让命中情况第一次
成为客户端可以直接对账的量。但 cache 按 token 前缀逐字节匹配，历史
只要被改写一次，改写点之后的前缀全部作废。于是 context 管理的两条
路线天然互相冲突：要控制 context 增长就得改写历史（compaction），
要保住 cache 就不能碰它。

我们做的事情是把这对矛盾放到一个尽可能干净的环境里测量。基座选了
一个很薄的开源 harness（pi）：session append-only，每次请求从同一
份历史确定性地重建，实测多轮 session 的 cache hit 稳定在 83–97%——
也就是说基座本身不制造 cache 问题，剩下的变量都是我们自己引入的。
compaction 以 extension 的形式挂在发送前的 context hook 上，核心是
一个纯函数投影：持久化的历史永不改写，只在发送时把窗口外的大块
tool result（read 的文件原文、bash 输出、edit 的参数）替换成结构化
摘要——路径、内容 hash、行数、首尾片段。投影是确定性的、幂等的，
所以第一次触发之后，被压缩区域重新做到 byte-stable，前缀可以再次
被 cache 命中；threshold 门控让触发尽量延后，门内保持原样发送。
这个设计押的是一个行为假设：模型读完文件后会把语义写进随后的叙述，
原文降格为「凭 hash 可再取」的引用是近似无损的。

Dogfooding 里最有价值的不是压缩率，是几类行为观察。

第一类是压缩后的重读。分析下来病因不同：有的是摘要结构太薄，模型
缺少判断「是否需要原文」的定位信息；有的是任务本身依赖逐字原文
（大规模机械改写就是典型），摘要注定不够——后者不是缺陷而是方法
的适用边界。两类需要的对策完全不同，前者改摘要结构，后者靠识别
工作负载类型。

第二类更细：模型改完文件后，在 context 里看到残留的改前内容，
会推断「刚才的 edit 静默失败了」，随后反复重读自证，表现为
非压缩内容的重读、cache hit 崩塌、reasoning token 激增。直觉的
修法是在 system prompt 里声明「你的修改是可靠的」，但这句不能写——
edit 确实可能失败，无条件的信任声明会制造幻觉。可行的修法是只注入
可机械验证的事实：edit result 在任何投影下保留路径、新 hash 与
diffstat；每份摘要携带取材时刻的内容 hash；发送时检测到 context 中
的视图 hash 落后于当前文件，就在 volatile 尾部追加一行确定性的提示
（「该视图早于你在第 N 轮的编辑，hash X→Y，需要现状请重读」），
只进当轮请求，不入持久历史，不动前缀。实测有效：模型据此只重读
真正过期的文件，对自己是最后编辑者的文件（hash 一致）不再重读，
提示行本身不影响 cache hit。这件事的一般化表述是：对无状态的模型，
信任只能建立在可核对的溯源上，不能建立在断言上。hash 这类看似
cache 管线的配件，实际是行为层的基础设施。

第三类是意外获得的对照。我们用来监督实验的 coding agent 自身的
context 满了，被它的框架用 LLM-summary 方式压缩——即被摘要成散文。
随后它出现了轻度的工作语义丢失：重复核查已完成的事项、对既有实现
的状态做出需要重新验证的断言。恢复靠的是仓库本身：任务契约、
git 历史、测试套件都在文件系统里，照此重建状态即可。有损摘要与
确定性投影的差异，在这个案例里是第一手的。

未决的是净省问题，也是最重要的问题。compaction 每次触发都砸掉一次
前缀连续性；省下的 input token 与放弃的 cache 折扣孰大，初步观察是
双模态的：干净的 read-heavy 分析里，压缩触发后无重读，cache hit
经历一次 dip 后随投影的 byte-stable 特性回升，账面倾向为正；歧义的
edit-heavy 工作里，重读可以把节省吃回去。也就是说答案是任务类型上
的分布，不是一个数。配对的受控对比（同任务、同模型、compaction
on/off，含 cache 折扣折算的总成本）还没有跑完，在那之前我们不引用
任何节省率数字。

方法上留下来的东西比结论多：历史 append-only、投影纯函数化、摘要
带 hash 溯源、失效模式按行为指纹分类并在 telemetry 里机械可判——
这些让 compaction 从一个「感觉更省」的技巧变成可证伪的实验对象。
